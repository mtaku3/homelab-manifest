# Coder GitHub External Auth — Design

## Goal

Enable GitHub as an external auth provider in Coder so that workspaces can
clone and push to private GitHub repositories under `github.com/mtaku3` using
OAuth credentials managed by Coder. This is independent of login auth (which
remains Google OIDC).

## Background

Coder is deployed via Helm + Kustomize at `dev/coder/`. Login auth uses Google
OIDC; the GitHub *login* OAuth provider is intentionally disabled
(`CODER_OAUTH2_GITHUB_DEFAULT_PROVIDER_ENABLE=false`). External auth is a
separate Coder feature for issuing per-workspace git credentials and uses
`CODER_EXTERNAL_AUTH_<n>_*` env vars.

## Configuration

A GitHub OAuth App has been created out-of-band by the user. Its callback
URL must be:

```
https://coder.mtaku3.com/external-auth/primary-github/callback
```

The provider is exposed inside Coder with these properties:

| Property      | Value             |
| ------------- | ----------------- |
| ID            | `primary-github`  |
| Type          | `github`          |
| Regex         | `github\.com/mtaku3` |

The regex restricts which git remotes trigger this auth provider, so only
repos under `github.com/mtaku3` will prompt workspace users to authorize.

## Changes

### 1. New SealedSecret

Create `dev/coder/github-external-auth-secret.yaml` containing the OAuth App
client ID and secret as a SealedSecret named `coder-github-external-auth` in
the `coder` namespace, with keys:

- `CODER_EXTERNAL_AUTH_0_CLIENT_ID`
- `CODER_EXTERNAL_AUTH_0_CLIENT_SECRET`

The secret is generated by the user via:

```bash
kubectl create secret generic coder-github-external-auth \
  --namespace=coder \
  --from-literal=CODER_EXTERNAL_AUTH_0_CLIENT_ID=<id> \
  --from-literal=CODER_EXTERNAL_AUTH_0_CLIENT_SECRET=<secret> \
  --dry-run=client -o yaml \
  | kubeseal -o yaml > dev/coder/github-external-auth-secret.yaml
```

### 2. `dev/coder/values.yaml`

Append five env vars to `coder.env`:

```yaml
- name: CODER_EXTERNAL_AUTH_0_ID
  value: "primary-github"
- name: CODER_EXTERNAL_AUTH_0_TYPE
  value: "github"
- name: CODER_EXTERNAL_AUTH_0_REGEX
  value: "github\\.com/mtaku3"
- name: CODER_EXTERNAL_AUTH_0_CLIENT_ID
  valueFrom:
    secretKeyRef:
      name: coder-github-external-auth
      key: CODER_EXTERNAL_AUTH_0_CLIENT_ID
- name: CODER_EXTERNAL_AUTH_0_CLIENT_SECRET
  valueFrom:
    secretKeyRef:
      name: coder-github-external-auth
      key: CODER_EXTERNAL_AUTH_0_CLIENT_SECRET
```

### 3. `dev/coder/kustomization.yaml`

Add `github-external-auth-secret.yaml` to the `resources:` list so kustomize
applies the SealedSecret alongside the helm release.

## Deployment

ArgoCD auto-syncs `dev/coder/`. Once the changes are committed and pushed via
PR, ArgoCD will roll the Coder deployment with the new env vars and the
SealedSecret will be unsealed by the cluster's sealed-secrets controller.

## Verification

After sync:

1. Coder pod should be `Running` with no env-var-related crash loops.
2. In the Coder UI, navigate to Account → External Authentication; the
   `primary-github` provider should appear and allow connecting.
3. In a workspace template referencing
   `data.coder_external_auth.primary-github`, the access token should be
   available as `CODER_EXTERNAL_AUTH_PRIMARY_GITHUB_ACCESS_TOKEN`.

## Out of Scope

- Changes to login auth (Google OIDC stays as-is).
- Workspace template updates to consume the new auth — templates are managed
  separately.
- Additional providers (GitLab, Bitbucket, etc.).
